Teori Git & Github (Catatan)
 
1. Git & Github adalah suatu hal yang berbeda 
2. Git Adalah VCS (version control system) Atau source code management 
sistem yang mengelolah perubahan dari sebuah dokumen 
3. memungkinkan bekerja berkolaborasi dengan lebih baik & mengetahui yang melakukan dan kapan
Sebuah perubahan terjadi 
4. memungkinkan kembali kekeadaan sebelum perubahan
file di dalam folder (repository / repo) 
5. riwayat perubahan file disimpan menggunakan rangkaiyan commit

================================================================================================================

GitHub Adalah Layanan cloud untuk menyimpan & mengelola project / repo git.
        Singkatan2 / Istilah2 Dalam GitHub
	  repo : folder project 
	  commit : rekaman dari repository
	  hash : penandaan unik pada sebuah commit 
	  checkout : berpindah kesebuah commit 
	  branch : cabang bebas dari sebuah commit
	  merge : menggabungkan brach
	  marge Conflict : baris yang sama di ubah oleh 2 brach yang berbeda
	  remote : sumber yang memiliki repo 
	  clone : mengambil repo dari remote
  	  push : mengirim commit ke repo
	  pull mengambil commit dari repo
	  
- Brancing / Brach Jalur 'Development' bebas sebuah commit (Fungsi)
	Membuat Git Branch 
	Membuat Snapshot tanpa mengnaggu jalur utama (Master brach)
	Fitur Experimental 
	2 Orang mengerjakan repo yang sama
	
- Fork / Forking (Fungsi)
       Membuat 'copy/duplikat dari repo
       orang lain (beserta history.
       Jembatan antara repo original dan duplikat nya.
       modifikasi terhadap repo original.
       berkontribusi terhadap repo orang lain.
